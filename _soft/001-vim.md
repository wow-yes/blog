syntax on：这可以确保 Vim 知道我在使用什么语法。做笔记、写这种文章时我主要使用 Markdown； 而在写小说的时候，纯文本是我的首选格式。
set noerrorbells：为了你的精神状态，我强烈建议打开这个选项。

你会注意到，我没有花更多时间讨论其他一些基本配置选项，因为我并不觉得那些对写作者来说有多重要。因为我做一些业余的编码工作，所以我的.vimrc配置反映了这一点。如果你只想在Vim上写作，那么上述配置就应该能让你顺利开始。

## Vim 范围选择

使用 Vim，你可以轻松地对文件中的一系列行执行操作。你可以通过起始行号、逗号和结束行号（包括）来指示范围。除了文字行号之外，你还可以使用句点（`.`）表示当前行，使用美元符号 （`$`） 表示文件缓冲区中的最后一行，以及使用百分号（`%`）表示整个文件。

这里举几个例子来说明。

- `:2,10d` 删除第 2 到 10 行
- `:25,$d` 删除从第 25 行到文件末尾的每一行：

你可以使用以下命令删除每一行： `:%d`

要将第 5 到 10 行复制（或转移）到第 15 行之后：

1. `:5,10t 15`

要将第 5 行到第 10 行移动到第 15 行之后（而不是复制）：

1. `:5,10m 15`

## 查看 VIM 启动时间

1. 使用内置命令生成启动日志

打开你的终端，执行以下命令。这条命令会启动 Vim，记录详细的启动时间到 /tmp/vim.log 文件，然后立即退出 。

```bash
vim --startuptime /tmp/vim.log +qall
```

2. 分析日志，找到耗时最长的项目

日志文件生成后，你需要从中找出最耗时的部分。最简单的办法是按耗时（第二列）进行排序，这样最慢的项目就会排在最后，一目了然 。

执行下面的命令，它会排序并显示日志内容：

```bash
sort -n -k 2 /tmp/vim.log
```

如果你想只看最耗时的前几名（例如前10名），可以这样：

```bash
sort -n -k 2 /tmp/vim.log | tail -n 10
```

3. 如何阅读和分析日志

你需要关注的是日志中的第二列（数字），它代表了每个环节消耗的时间，单位通常是毫秒 (ms) 。

快速判断标准：如果启动总时间在 200ms 以下，体验会非常流畅；超过 500ms 就会感觉到卡顿；如果超过 1秒，就需要认真优化了 。

找出罪魁祸首：排在末尾的那些行，就是加载最慢的环节。它们通常是某个具体的插件脚本或你的 .vimrc 中的某段配置 。例如，如果看到某个以 nerdtree 或 syntax 开头的脚本耗时很长，就知道问题所在了。

在vim中设置viminfo的大小

```vim
" 完全控制 viminfo 存储的内容
set viminfo='100,<50,s10,h,%

" 参数说明：
" '100  - 最多保存 100 个文件的标记（不是历史命令！）
" <50   - 每个寄存器最多保存 50 行
" s10   - 寄存器大小超过 10KB 就不保存
" h     - 禁用高亮搜索的历史记录（hlsearch）
" %     - 保存/恢复缓冲区列表
" !     - 保存/恢复全局变量（以大写字母开头的）
" /     - 保存搜索历史的最大条数（未指定时默认50）
" :     - 保存命令历史的最大条数（未指定时默认50）
" @     - 保存输入行历史的最大条数（未指定时默认50）

" 更激进的限制（如果还想进一步压缩）：
" set viminfo='20,<10,s1,h
```
